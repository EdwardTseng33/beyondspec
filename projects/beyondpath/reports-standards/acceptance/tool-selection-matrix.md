> **一句話**：MBA 工具不是並列菜單，是 **PATH 四維的增強武器**——選擇、融合、翻譯，缺一不可

## 零、戰略核心宣言（Howl's Law）

> **PATH 不是另一套市場分析框架，PATH 是「為小團隊早期創業者設計的決策引擎」。**
> **MBA 工具（SWOT / PESTEL / 五力 / STP / 安索夫 / SMART / JTBD）是支援 PATH 的內部零件，不是對外展示的標題。**
> **讀者永遠看到 P/A/T/H 四個字母，永遠不應該在 hero 看到「本報告採用 PESTEL 分析」這種教科書敘事。**

**Howl's Law 三條**：
1. **融合 > 並列**：工具要翻譯成 PATH 維度的子項，不能獨立成章
2. **讀者情境決定工具**：5 份報告面向 5 種讀者，不是同一套菜單
3. **創業者本人不需要學 MBA**：工具隱身在後台，結論顯身於前台

## 一、5 × 5 工具選擇矩陣

**欄**：5 份報告 / **列**：5 個 MBA 工具 + 2 個原生工具
**格值**：✅ 必用 | 🟢 選用 | 🟡 慎用（需簡化）| ❌ 不用

| 工具 \ 報告 | PATH 診斷 | Lab 分析 | 市場探測 | 市場報告 | AI 問卷 |
|---|:---:|:---:|:---:|:---:|:---:|
| **SWOT** | ❌ | ❌ | 🟡 | ✅ | ❌ |
| **PESTEL** | ❌ | ❌ | 🟡（簡化 PEST）| ✅ | ❌ |
| **波特五力** | 🟢（借 T 維）| ❌ | 🟢（簡化版）| ✅ | ❌ |
| **STP** | ✅（融入 A 維）| 🟢（Persona=S）| 🟡 | ✅ | ❌ |
| **安索夫矩陣** | ❌ | ❌ | ❌ | 🟢 | ❌ |
| **SMART**（原生）| ✅ | ✅ | ✅ | ✅ | ✅ |
| **JTBD**（原生）| ✅（融入 P 維）| ✅ | ❌ | 🟢 | 🟢 |

### 1.1 逐格判定理由

**PATH 診斷欄（最嚴格）**——讀者是創業者本人，2 分鐘做決定：
- ❌ SWOT：inside-out 哲學衝突；H 維已涵蓋資源盤點
- ❌ PESTEL：太宏觀，早期用不到 5 年視角
- 🟢 五力：**只借「產業競爭強度」一格**進 T 維
- ✅ STP：**完全融入 A 維** — A 就是 STP
- ❌ 安索夫：成長策略是 PMF 後才有用
- ✅ SMART + JTBD：**融入 H 維 + P 維**

**市場報告欄**——投資人 / 高管讀者，30-60 分鐘完整分析：
- ✅ **這才是工具大集合的正確場域**（讀者期待嚴謹戰略分析）
- 但必須用 McKinsey Pyramid 串聯（見 §4）

**AI 問卷欄**——問卷設計者讀者：
- ❌ 宏觀工具全棄
- 🟢 JTBD 用在「題目驗證什麼假設」
- ✅ SMART 用在「發送後成功指標」

### 1.2 Less Is More 極致原則

- **PATH 診斷 + AI 問卷 = 近乎零 MBA 工具露出**（只 SMART + JTBD 隱身）
- **市場報告 = 5 工具全集合**（但用金字塔收束）
- **Lab / 探測 = 只選 1-2 工具強化特定章節**

## 二、PATH 四維 × MBA 工具融合表（**核心靈魂**）

| PATH 維 | 原定義 | **融合後增強版定義** | 融入工具 | 讀者感知 |
|---|---|---|---|---|
| **P** Problem | 問題是否值得解？ | **你服務對象真的把這件事當痛點嗎？**（JTBD + 痛點真實性 + 3 維量化：頻率/強度/現有替代方案不滿度）| JTBD（主軸）+ SMART | 「我懂我要解決什麼」 |
| **A** Audience | 你知道在跟誰說話嗎？ | **你能用一句話描述他，他會認同這就是他嗎？**（STP 三步驟：Segmentation → Targeting → Positioning）| STP（完全融合）+ JTBD（互證） | 「我描述的人就是我」 |
| **T** Traction | 有人在買單嗎？ | **市場用什麼訊號告訴你「有戲」？**（waitlist / LOI / 付費 / 推薦擴散 + 簡化五力評估產業競爭強度）| 五力（借「競爭強度」單格）| 「我知道為什麼會有人買」 |
| **H** Hits-Execution | 你能把它做出來嗎？ | **三個月內你真能跑完 MVP → 收第一筆錢嗎？**（資源盤點：人/錢/時間/能力缺口 + SMART 化下一步）| SMART（驗收所有 action）+ 類 SWOT 的 S/W | 「我知道下一步做什麼」 |

### 2.1 P 維 · JTBD 融合公式
```
P 分數（0-25）=
    痛點真實性（10 分）· 訪談/問卷樣本強度
  + 痛點頻率（5 分）· 每週/月/季發生
  + 現有替代方案不滿度（5 分）· 競品/土法煉鋼痛苦程度
  + JTBD 清晰度（5 分）· 能否一句話說「他雇用產品幹嘛」
```

**正例**：「P 18/25。你的用戶在僱用 BeyondPath 來『一個人把驗證到營收跑完』——75% 訪談者引用關鍵字『一個人』。痛點頻率每週發生，現有替代方案（Notion + Excel + Line）滿意度 2.3/5。」

**反例**：「P 18/25，痛點明確」→ 並列式敘述 FAIL

### 2.2 A 維 · STP 融合公式
```
A 分數（0-25）=
    Segmentation 明確度（8 分）· 切幾群？依據合理嗎？
  + Targeting 聚焦度（8 分）· 選哪一群？為何是這群？
  + Positioning 差異化（9 分）· 一句 statement 能否和競品區分
```

**Positioning Statement 強制模板**：
> For **[who]** who **[pain/need]**, **[product]** is a **[category]** that **[key benefit]**, unlike **[alternative]** which **[alternative weakness]**.

**BeyondPath 自己跑 A 軸範例**：
> For **台灣 1-20 人小團隊創辦人** who **一個人扛驗證到營收整條鏈**, **BeyondPath** is a **AI 商業引擎** that **讓 Claude 做苦工、你做決策**, unlike **Notion + 拼裝工具** which **只給空白畫布不給流程**.

### 2.3 T 維 · 簡化五力單格
```
T 分數（0-25）=
    內部訊號（15 分）· waitlist / LOI / 付費客戶 / 推薦
  + 外部壓力（10 分）· 簡化五力「產業競爭強度」
```

**外部壓力簡化五力（10 分）**：

| 五力維 | 早期創業者版本問題 | 分數 |
|---|---|:---:|
| 現有對手 | 台灣有幾個直接競品？成熟度多高？ | 0-2 |
| 新進入者 | 這賽道進入門檻如何？ | 0-2 |
| 替代品 | 用戶現在用什麼土法煉鋼？難取代嗎？ | 0-2 |
| 買方議價 | 用戶付費意願 + 價格敏感度 | 0-2 |
| 供應方議價 | 依賴哪些 API/資料源？被綁架風險 | 0-2 |

**哲學**：不讓創業者跑去查 Porter 1980 原文；讓他 30 秒答完 5 問題。

### 2.4 H 維 · SMART + 輕量 S/W
```
H 分數（0-25）=
    資源盤點（10 分）· 人/錢/時間/能力缺口自評
  + SMART 化下一步（15 分）· Top 3 actions 的 SMART 合規度
```

**SMART 在 BeyondPath 語境**：

| 字母 | 原意 | BeyondPath 版 |
|---|---|---|
| **S**pecific | 具體 | 動詞開頭 · 名詞具體 · 不准「加強 marketing」 |
| **M**easurable | 可衡量 | 必含數值或布林判準 |
| **A**chievable | 可達成 | 工時 ≤ 2 週 · 資源現有 |
| **R**elevant | 相關 | 必須回指某個 PATH 軸的弱項 |
| **T**ime-bound | 有時限 | 含截止日或「2 週內」 |

## 三、讀者 → 工具 對應表

> **一個工具能不能用，取決於讀者在那個情境下有沒有耐心讀。**

| 報告 | 讀者 | 情境 | 時長 | 決策 | 工具容忍度 |
|---|---|---|:---:|---|:---:|
| **PATH 診斷** | 創辦人本人 | 手機 · 咖啡廳 · 快速檢查 | **2-5 分鐘** | Kill / Pivot / Persevere | **極低** |
| **Lab 分析** | 行銷 / PM | 桌電 · 週會前 | **10-15 分鐘** | 改哪句文案 / 調 persona | **低** |
| **市場探測** | 想進場者 | 桌電 · 週末研究 | **15-20 分鐘** | niche 能否進？ | **中** |
| **市場報告** | 投資人 / 高管 | 桌電 · 正式提案 | **30-60 分鐘** | 投資 / 進入 / 擴張？ | **高** |
| **AI 問卷** | 問卷設計者 | 桌電 · 設計中 | **5 / 30 分鐘** | 這題有偏差嗎？ | **極低** |

### 3.1 工具容忍度決策樹
```
Q1：讀者是決策者本人嗎？
├── 是 → Q2
└── 否（下屬/分析師）→ 容忍度可到「中高」

Q2：讀者做「快速判斷」還是「深度投資」決策？
├── 快速判斷（< 15 分鐘）→ 容忍度「極低」，全部融合隱身
└── 深度投資（> 30 分鐘）→ 容忍度「高」，可顯式章節

Q3：讀者需要把報告拿給第三方看嗎？
├── 是（投資人 / 董事會）→ 工具露出可「支撐權威感」
└── 否（自用）→ 工具越少越好，只要結論
```

## 四、市場報告 · McKinsey Pyramid 串聯架構

> 5 工具全集合 ≠ 5 章並列。必須**一句 thesis 領軍、金字塔收束**。

```
                    ┌──────────────────────────┐
                    │  §1 THESIS（頂層 · 一句話）│
                    │  「未來 18 個月，贏家屬於    │
                    │   對小團隊設計的 category    │
                    │   definer。」              │
                    └──────────────────────────┘
                                │
              ┌─────────────────┼─────────────────┐
              ▼                 ▼                 ▼
       ┌──────────┐      ┌──────────┐      ┌──────────┐
       │ §2 DEMAND │      │ §3 SUPPLY │      │ §4 TIMING │
       │  需求支柱  │      │  供給支柱  │      │  時機支柱  │
       │ PESTEL S/E│      │ 五力+安索夫│      │ STP+趨勢線 │
       └──────────┘      └──────────┘      └──────────┘
                                │
                    ┌──────────────────────────┐
                    │  §5 SWOT 戰略總結章        │
                    └──────────────────────────┘
                                │
                    ┌──────────────────────────┐
                    │  §6 ROADMAP + EXIT        │
                    │  0-3M / 3-12M / 12M+ SMART│
                    └──────────────────────────┘
```

### 4.1 寫法紀律：工具名隱身 ≠ 內容偷工

**❌ 錯誤**（教科書式）：
> ## §2.1 PESTEL 分析
> **P 政治**：台灣政策如何 ...

**✅ 正確**（BeyondPath 式）：
> ## §2 為什麼現在是小團隊 AI 工具的起點
>
> 三件事同時發生在 2025-2026：政府 AI 補助落地（政策）、LLM API 價格一年降 8 成（經濟）、台灣一人公司數量 YoY +34%（社會）、Claude Agent SDK 讓非工程師也能編排 AI（技術）。這四條趨勢線在 2026 Q2 首次交會——前面 8 年不可能，2027 以後擁擠。
>
> 📐 本段分析架構：PESTEL 四軸，詳細支撐見附錄 A。

**差別**：工具隱身結論之後，細節放附錄。讀者先拿 insight，再拿 method。

## 五、不要全用的 5 個戰略理由

### 理由 1 · 認知負擔定律
工作記憶 7±2 chunks。同時塞 5 框架 = 讀者大腦爆炸 = 讀完沒結論留下。Top-player（Maze/Dovetail/Productboard）共同默契：每份報告**1 主導框架 + 2-3 輔助**。

### 理由 2 · 品牌差異化定律
讀者讀完說：
- 「哦這是 SWOT + PESTEL 的報告」→ **BeyondPath 已死**
- 「哦這是 PATH 四維的報告」→ **Category 成立**

**Category creation > Analysis completeness，100 倍以上**。

### 理由 3 · 決策速度定律
早期創業者最稀缺的不是分析工具，是**時間 + 注意力**。多給一個工具 = 多給一個延遲決策的藉口。BeyondPath 價值主張是「跑得動」，不是「想得全」。

### 理由 4 · 工具選擇的意義 > 工具使用完整性
**展現品味的不是「你用了多少工具」，是「你為何不用某工具」**。
敢說「本報告不採用安索夫矩陣，因為早期驗證階段不在『市場滲透 vs 新市場』的決策象限」——比硬塞進去更顯戰略成熟。

### 理由 5 · Top-player 實證
| 產品 | 主導框架 | 拒絕露出的工具 |
|---|---|---|
| Maze | Usability benchmark | 不露 SWOT / PESTEL |
| Dovetail | Insight cluster + Quote | 不露任何 MBA 模型 |
| Productboard | Top 3 this week | 不露 SWOT / 安索夫 |
| McKinsey | 3 pillars thesis | 工具全放附錄 |
| YC RFS | Hypothesis + Confidence | 拒絕框架敘事 |

**結論**：Top-player 全部**隱藏工具、突顯洞察**。

## 七、霍爾個人戰略判斷：只留 3 個工具

### 7.1 Top 3
1. **JTBD**（Jobs-to-be-Done）
2. **STP**（Segmentation / Targeting / Positioning）
3. **SMART**

### 7.2 為何是這 3 個

**JTBD 第一名**——它回答「為什麼有人要買」創業最根本的問題。沒有 JTBD 就沒有 Problem 的合法性。Christensen「買電鑽不是要電鑽，是要牆上的洞」比任何 MBA 框架更接近 BeyondPath「一個人把驗證到營收跑完」的靈魂。

**STP 第二名**——讓「你在跟誰說話」變成可操作的三步驟。BeyondPath 對外溝通最缺的就是一句 positioning statement。沒有 STP，A 軸只是「目標用戶是 1-20 人小團隊」這種無效描述。

**SMART 第三名**——所有 Next Action 的驗收標準。BeyondPath 終極差異化是「跑得動」，SMART 是「跑得動」的可驗證定義。沒有 SMART，「今天可以做一件事」退化成「加油雞湯」。

### 7.3 為何不是 SWOT / PESTEL / 五力 / 安索夫

| 淘汰 | 主因 |
|---|---|
| **SWOT** | 太泛太舊太多人用 = 無差異化；inside-out 與 outside-in 哲學混雜 |
| **PESTEL** | 5-10 年戰略視角，和 BeyondPath「2 週內下一步」節奏衝突 |
| **波特五力** | 對成熟產業強，對新 category 限制想像；只留「競爭強度」單格夠 |
| **安索夫** | PMF 後才有意義；90% 用戶還在 Problem 驗證階段 |

### 7.4 核心戰略意涵

**JTBD + STP + SMART = validate → target → execute 完整閉環**

這三個工具共同特徵：**以動詞開頭、以可驗證結果收尾**。
這是 BeyondPath 靈魂——**不做思考框架，做執行引擎**。

**淘汰 4 工具共同特徵**：**以名詞收尾、以象限呈現**。
象限是**分類學**，不是**決策學**。
BeyondPath 是**決策學**產品，不是分類學產品。
