## 一、商業模式設計

### 1.1 核心商業模式結構

**定位公式（配合 Edward 的成本優勢）**：

Edward 的核心競爭優勢——「AI native 1-3 人小團隊、開發/人力成本遠低於大牌」——對商業模式的直接含義是：

| 傳統 SaaS 大牌 | BeyondSpec 的結構優勢 |
|---|---|
| 工程師團隊 20-100 人 → 功能更版慢 | AI native 3 人 → 可快速出模組 |
| 人力成本高 → 定價被迫高 | 人力成本低 → 更低價打進市場 OR 更高毛利 |
| 客製需要大量工程資源 → 拒絕 | 客製成本低 → 可做大牌不願做的案 |
| 本地化是額外工程量 | 原生中文台灣在地訊號是 DNA |

這個優勢轉成商業模式的含義：**用比競品低 30-50% 的定價切入，但維持 70-75% 以上毛利（靠低結構成本）**。這不是折扣策略，是結構性定價護城河。

### 1.2 訂閱結構設計（三層 + 按量制）

**推薦結構：「基礎訂閱 + AI 用量分層」混合模型**

2026 年 SaaS 定價趨勢：43% 公司已在「訂閱+用量」混合模式，預計 61% 覆蓋率（Chargebee 2025 State of Subscriptions）。BeyondSpec 的模組化天然適合這個方向。

**四層訂閱結構（建議）**：

| 方案名稱 | 定位 | 核心內容 | 建議定價 |
|---|---|---|---|
| **Starter（試用真功能）** | 個人 / 一人工作室 | 任務看板 + 客戶 CRM + 基礎報價 + PATH 診斷（限 3 次/月）| **NT$0/月（永久免費）** |
| **PRO（跑起來）** | 1-5 人小團隊 | 全模組解鎖 + AI 用量 50 次/月 + 完整 PATH 工具 + 報價/收款閉環 | **NT$499/月（或 NT$4,990/年）** |
| **MAX（長大了）** | 5-20 人團隊 | PRO 全功能 + AI 用量 200 次/月 + 戰情室 + 算薪出缺勤 + 優先支援 | **NT$999/月（或 NT$9,990/年）** |
| **Team（協作版）** | 多人共用帳號 | MAX 功能 + 每增加 1 人 NT$299/人/月 + 管理員權限 + 分組 | **NT$999 + NT$299/人** |
| **Enterprise** | 10 人以上 / 客製需求 | 客製模組 + 專屬客服 + SLA + 本地部署評估 | 洽談（建議底線 NT$3,000/月起）|

**AI 用量超出後的選擇**（解決毛利保護問題）：

| 超用方案 | 設計 | 目的 |
|---|---|---|
| 自動封頂 + 通知 | 超過月額度後 AI 功能暫停，其餘功能正常 | 用戶不會意外被扣費 |
| 加購 AI Pack | NT$199/100 次（估算每次 API 成本約 NT$5，售價 NT$2，毛利 60%）| 轉換率驅動器 |
| 升級 | 推薦直接升方案（年繳便宜 17%）| 年化收入提升 |

### 1.3 模組加購 vs 全包決策

**建議：全包為主，模組加購作為選配（不強迫）**

理由：
1. 台灣小企業不善於自己組合功能，複雜選擇增加決策成本 → 轉換率下降
2. 全包簡化銷售敘事（「一個平台，全部搞定」比「你需要選 A+B+C」好說）
3. BeyondSpec 的競爭優勢之一是「閉環」，割裂模組就破壞這個敘事

**但以下模組可考慮收費分級**（因為邊際成本高）：

| 模組 | 為什麼可按量/加購 | 建議設計 |
|---|---|---|
| Lab 合成訪談 | 每次合成調用 Claude API，成本 NT$5-20 | PRO 包 5 次/月，超用 NT$49/次 |
| 市場報告 | 長篇研究，API 消耗最大 | PRO 包 2 份/月，超用 NT$99/份 |
| 問卷引擎（AI 分析） | 分析需要大量 token | PRO 包 10 份/月，超用 NT$29/份 |
| 算薪出缺勤 | 員工數量線性增加成本 | 按員工人數收費，10 人以下包含，10+ 每人 NT$99 |

### 1.4 Freemium vs 免費試用決策

**建議：永久免費 Starter（有限制的 Freemium）**，而非限時試用

2026 基準數據：
- 免費試用（需信用卡）轉換率 31.4%
- Freemium 轉換率 8.9%（全球均值）
- 但 Freemium 獲得的 **免費用戶池更大**，台灣市場信任建立期更長

對 BeyondSpec 的建議邏輯：台灣 SaaS 付費習慣未完全成熟（付費滲透率 20-30%），Freemium 是**信任建立工具**，比限時試用更適合台灣。台灣用戶傾向「用了再說，用好了再付」。

**Freemium 護欄設計（不破壞升級動機）**：

| 功能 | Starter 免費版限制 | PRO 解鎖 |
|---|---|---|
| PATH 診斷 | 3 次/月 | 無限 |
| Lab 合成訪談 | 0（需升級）| 5 次/月 |
| 報價單 | 5 份/月 | 無限 |
| 客戶 CRM | 20 個聯絡人 | 無限 |
| 任務看板 | 2 個看板 | 無限 |
| AI 用量 | 0（體驗後封鎖）| 50 次/月 |
| 資料匯出 | 不可 | 可 |
| 戰情室 | 不可 | PRO+ |

**關鍵設計**：讓 Freemium 用戶**感受到 AI 的力量**（可以試用一次 PATH 診斷）→ 然後碰到「你已用完本月配額」→ 升級動機最強。

### 2.2 AI API 成本護欄（毛利保護核心）

這是 BeyondSpec 財務健康的最大變數。現在算清楚。

**Claude API 2026 年定價**（已驗證）：
- Haiku 4.5：$1.00 input / $5.00 output（每百萬 token）
- Sonnet 4.6：$3.00 / $15.00
- Opus 4.7/4.8：$5.00 / $25.00

**BeyondSpec 功能的 token 估算**（每次 AI 調用）：

| 功能 | 模型選擇 | 估算 token/次 | 成本/次 (USD) | 成本/次 (NT$) |
|---|---|---|---|---|
| PATH 診斷 | Sonnet 4.6 | 5,000 input + 2,000 output | $0.015 + $0.03 = $0.045 | NT$1.44 |
| Lab 合成訪談（1 組問題） | Sonnet 4.6 | 8,000 input + 4,000 output | $0.024 + $0.06 = $0.084 | NT$2.69 |
| 市場報告生成（完整版） | Sonnet 4.6 | 20,000 input + 10,000 output | $0.06 + $0.15 = $0.21 | NT$6.72 |
| 問卷 AI 分析 | Haiku 4.5 | 3,000 input + 1,000 output | $0.003 + $0.005 = $0.008 | NT$0.26 |
| 客戶 AI 洞察 | Haiku 4.5 | 2,000 input + 500 output | $0.002 + $0.0025 = $0.0045 | NT$0.14 |

**PRO 方案 AI 成本結構估算（每月每用戶）**：

| 假設用量 | 計算 | 月 AI 成本 |
|---|---|---|
| 保守（低度使用）| PATH 診斷 2 次 + 報告 1 份 + 問卷 5 份 | NT$2.88 + NT$6.72 + NT$1.3 = **NT$10.9** |
| 中度使用 | PATH 5 次 + 訪談 3 次 + 報告 2 份 + 問卷 10 份 | NT$7.2 + NT$8.07 + NT$13.44 + NT$2.6 = **NT$31.3** |
| 重度使用（上限 50 次）| 全功能拉滿 | 估 **NT$60-80** |

**PRO NT$499 毛利結構**：

| 項目 | 保守場景 | 中度場景 | 重度場景 |
|---|---|---|---|
| 訂閱收入 | NT$499 | NT$499 | NT$499 |
| AI API 成本 | NT$11 | NT$31 | NT$80 |
| 基礎設施（Firebase/Vercel 分攤）| NT$10 | NT$10 | NT$10 |
| 客服/支援分攤 | NT$15 | NT$15 | NT$15 |
| **毛利** | **NT$463 (93%)** | **NT$443 (89%)** | **NT$394 (79%)** |

**結論**：PRO NT$499 在重度使用場景下毛利仍有 79%，遠高於傳統 SaaS 的 60-70% 標準。這是因為 Edward 的低人力結構成本支撐。

**護欄設計**（防止毛利崩潰）：
1. **月額度限制**：PRO 50 次 AI 調用封頂是對的，防止無限成本
2. **Haiku vs Sonnet 路由**：非核心 AI 功能（問卷分析、小洞察）用 Haiku（5 倍便宜），核心分析（PATH 診斷、市場報告）用 Sonnet
3. **Prompt Caching**：重複的系統 prompt（用戶個人設定、工作空間描述）開啟 caching，省 90% input 成本
4. **Batch Processing**：非即時需求（市場報告生成）走 batch API，省 50%
5. **每月 AI 成本告警**：當某用戶 AI 成本 > NT$200 時觸發預警（可能是 API 被濫用 or 超重度）

## 三、市場打法（GTM）

### 3.1 先打哪個客群：排序與理由

**Y1 聚焦建議：先打「數位行銷代理商 + 設計工作室」雙軌**

| 客群 | 排序 | 理由 |
|---|---|---|
| 數位行銷代理商（10,000-18,000 家）| **#1 主攻** | ARPU 最高、同時用驗證+閉環、在 Inside/數位時代社群活躍、口碑傳播快 |
| 設計/創意工作室（20,000-40,000 家）| **#1 共推** | 體量大、任務看板+報價是剛需、IG/Behance 社群鏈密集 |
| B2B 顧問（8,000-15,000 家）| **#2 候補** | 付費意願強，但客群相對分散、較需要 1:1 銷售 |
| 科技新創（3,000-6,000 家）| **#3 長線布局** | PATH 驗證工具天然吻合，但需培育期 |
| 小型服務業（50,000-80,000 家）| **#4 後期** | 體量最大但 ARPU 低、獲客成本難壓縮 |

**為什麼不先打「體量最大的小型服務業」**：

台灣美容/餐飲/健身等服務業主的工具採用率低、AI 工具陌生、決策週期長、ARPU 低（NT$290-399）。Y1 資源有限時，應先打「付費意願強+ARPU 高+口碑傳播快」的族群，再從成功案例出發向外擴散。

### 3.3 PLG 漏斗設計（2026 最新模型）

2026 年 PLG 最佳實踐：58% 的 B2B SaaS 跑某種 PLG 模式，91% 計畫增加投入（OpenView 2024）。BeyondSpec 天然適合 PLG，因為產品功能本身就是最好的銷售工具。

**BeyondSpec PLG 漏斗（四層）**：

```
[曝光層] SEO / 社群 / 口碑 / 生態合作
    ↓
[試用層] Starter Freemium 永久免費 → 體驗 PATH 診斷 1 次 + 建任務看板 + 加 5 個客戶
    ↓
[啟動層] 觸碰「升級牆」→ 「已用完配額」/ 「加更多客戶需升級」/ 「想要 AI 市場報告需 PRO」
    ↓
[付費層] PRO NT$499/月 → 使用 6 個月後向 MAX 升級
    ↓
[倡導層] 分享工作空間 / 邀請隊友 / 在社群推薦 → 帶來新的曝光層流量
```

**啟動率（Activation）是最關鍵指標**（2026 SaaS 基準：只有 34% PLG 公司追蹤這個）：

定義 BeyondSpec 的「啟動」事件：
> 用戶在第一週內完成 PATH 診斷 1 次 + 新增客戶 3 個 + 建立 1 個任務 = **「Aha Moment」**

達到 Aha Moment 的用戶轉換率估計可達 30%+（行業基準，BeyondSpec 需 A/B 驗證）。

**關鍵轉換觸發設計**：

| 情境 | 觸發訊息 | 升級 CTA |
|---|---|---|
| 用完 PATH 診斷配額 | 「你已完成本月 3 次診斷。下個季度還有 2 個月——你的競品此刻也在用 AI 分析市場。」| 立即升級 PRO NT$499 |

### 4.3 財務風險與健康度

**風險一：AI API 成本上升（中等風險）**

- 緩解：月額度封頂 + Haiku/Sonnet 路由 + Batch/Caching 優化 → 即使 API 漲 50%，PRO 方案毛利仍 > 75%
- 預警線：當月 AI 成本 / 訂閱收入 > 15%，啟動定價檢討

**風險二：Freemium 用戶不轉付費（高風險，最需要持續監測）**

- 2026 全球 Freemium 轉換率均值 8.9%；台灣保守估 5-7%
- 目標：透過啟動漏斗設計提升至 10-12%
- 預警線：前 6 個月若轉換率 < 5%，立即做用戶訪談找 blocker

**風險三：高度依賴 Edward 個人（一人公司風險）**

- 這是 BeyondSpec 從顧問生意轉 SaaS 最關鍵的結構性轉變——SaaS 需要「產品本身」說話，不能依賴 Edward 親自售出每一個客戶
- 緩解：先做好 PLG 和自助升級，再考慮內容行銷系統化

**風險四：Notion / 大牌本地化加速（中等風險）**

- Notion 台灣中文化 + AI 進一步降價是 18-36 個月最大威脅
- 緩解：搶先佔「台灣在地訊號（PTT/Dcard/蝦皮）+ 中文報告 + 閉環」的認知定位
