## 排程（critical path）
- **Day 1（Edward·硬前置）**：① 綠界商店帳號（定期定額 + 電子發票權限）② 電子發票字軌（國稅局 / 綠界代辦）③ 確認統編 / 營業登記。蘇菲同步起草隱私政策 + 服務條款。
- **綠界審核期（城堡並行 ~7.5 人天）**：catalog 改價 + AI 額度權威化（Firestore + Worker 防作弊）+ Starter 7 天軟鎖 + 沙利曼把 webhook 驗章 / 取消停扣 / 發票格式驗證的防護先寫好。
- **金鑰到手**：接綠界定期定額 + 電子發票 + webhook（idempotent + cron 雙保險）→ PRO 隧道跑通 → 收第一筆錢。
- **v1（2–3 週）**：MAX 999 + 職務制選模組 + 年繳 + 90 天清資料 + 漏斗監測。
- **v2（4–6 週）**：Team seat 合併計費 + 逐人權限 + 客戶 portal/品牌露出（網路效應）。

## 三條紅線（suliman·收真錢前必過）
1. **webhook 必 server 驗章**（綠界 CheckMacValue）才解鎖，前端不能自解（否則偽造成功通知就白嫖）。
2. **「取消訂閱」必真呼叫綠界停扣**，不能只改自家資料庫（否則下月續扣 → 信用卡爭議）。
3. **隱私政策 / 條款 + 不可預設打勾同意**；已開發票 / 扣款憑證與產品資料**分離保存**（稅法 5–7 年），不隨 Starter 90 天清資料一起刪。

## 待 Edward 拍板（2 個決策）
1. **Starter：7 天時間硬鎖 vs 永久免費 + 功能配額鎖**？霍爾 + 卡西法 + 策略 v2 都偏永久免費（台灣信任期長、7 天可能在用戶感受到「追回一筆款」價值前就把人擋掉、反傷主訴求轉換）。建議至少軟鎖（能看不能編 + 可匯出）+ 7 天做成可調參數能 A/B。← 決定 ② 那塊怎麼做。
2. **PRO MVP：先固定核心模組組、職務制延 v1**？（避免「職務→模組矩陣」吃掉金流時間）建議是。
