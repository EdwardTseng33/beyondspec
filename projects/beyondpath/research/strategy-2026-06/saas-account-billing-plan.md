## 1. 帳號模型：以「工作空間」為單位
- 一個工作空間 = 一個生意/團隊。owner 註冊 → 自動建空間 → 訂閱方案掛在空間。
- 登入：Google / email（Firebase；app 已有 resolveWorkspace 自動建空間地基）。
- 模組可用性 = 方案決定；Team 再由 owner 對每個成員指派權限。

## 2. 四方案規格（Edward 拍板）
| 方案 | 對象 | 模組 | AI 額度 | 計費 |
|---|---|---|---|---|
| **Starter 免費** | 一人試水溫 | 全模組（Max 級） | 50 / 月 | **7 天試用，之後鎖定** |
| **PRO 499/月** | 跑起來 | **依職務選**（可選職稱）+ 專案與應用基礎模組 | 50 / 月 | 永久訂閱 |
| **MAX 999/月** | 長大了 | 全模組 | **1,000 / 月** | 永久訂閱 |
| **Team** | 工作室 / 中小企業 | 多人協作 + 匯入 + 應收款儀表板 + 優先支援 | 依每 seat（Pro/Max） | 主帳號合併計費 |

差異化三條線：① AI 用量（成本來源）② 人數 ③ 模組廣度（職務制 vs 全模組）。

## 3. Starter「7 天試用 → 鎖定」設計
- **Day 0**：免費註冊，**不需信用卡**（降摩擦、顧台灣信任）。立即開全模組（Max 級）+ AI 50。
- **Day 0–7**：完整體驗。
- **Day 5 / 6**：提醒「還有 X 天，升級就保留資料 + 繼續用」。
- **Day 7 未付 → 帳號硬鎖（Edward 2026-06-24 拍板）**：帳號綁定鎖死，**除非付費才能繼續使用該帳號**（非 read-only，是停用整個帳號）。
  - **資料保留不刪 → 付費即恢復**：owner 之後回來付費，帳號 + 資料原樣解鎖（硬前提：資料必須保留，否則「付費繼續使用」無從恢復）。
  - **90 天**仍未付 → 寄信預告 → 才清產品資料（個資法：不無限留存）；已開發票/扣款憑證另存不刪（稅法 5–7 年）。
  - 鎖定畫面：「試用結束，付費解鎖你的帳號」+ 單一付費 CTA。
- 蘇菲已提諫言（硬鎖 vs 策略 v2 永久免費的台灣轉換風險）→ Edward 聽過後拍板「硬鎖 + 帳號綁定」。執行照此；7 天門檻建議做成可調參數，未來要 A/B 不必改架構。

## 4. Team：多人 + 合併計費
- **主帳號（owner）**＝空間擁有者，持訂閱 + 計費。
- owner 邀請成員 → 每成員＝一個 **seat**，等級＝ PRO 或 MAX（owner 逐人選）。
- owner 指派每成員的模組權限（共同編輯 + 共享空間資料）。
- **帳單＝ owner 一張**：總額 = Σ(各 seat 的 PRO/MAX 月費)。例：owner Max 999 + 成員A Pro 499 + 成員B Pro 499 = **NT$1,997/月**。
- 加 / 減人 → 下個帳期調整總額。
- 一張電子發票開給 owner（公司戶填統編 + 抬頭）。

## 5. 金流 + 發票 + 付費流程（綠界 ECPay）
- **金流**：綠界**信用卡定期定額**（月繳 / 年繳自動扣款）。
- **發票**：綠界**電子發票** API——扣款成功自動開立。
  - 公司戶（多數客戶）：填**統編** + 抬頭 → 三聯式。
  - 個人戶：載具（手機條碼）/ 捐贈 → 二聯式。
- **付費流程**：app 內選方案 → 導綠界結帳頁（卡片授權，**app 不碰卡號**）→ 填統編 / 載具 → 授權成功 → 綠界 webhook 通知 app 解鎖方案 → 每月自動扣款 + 自動開發票 email。
- **Team**：owner 一個定期定額授權，金額＝總 seat 費；加人時改授權金額。
- **合規（Gate 5 沙利曼必驗）**：電子發票財政部規範（字軌 / 上傳）、卡號 PCI（綠界托管、app 不存卡號）、個資法（付款資料）。

## 6. 開發順序
1. **落地頁定價段 + 信任段**（現在·蘇菲建）：4 方案 + Notion 比價錨點 + 信任四件。CTA 誠實（免費開始 / Team 聯絡開通）。
2. **帳號地基**（app）：方案 + 模組權限 + AI 額度掛到工作空間；Starter 7 天 + 鎖定狀態。
3. **金流串綠界**：定期定額 + 電子發票（**前置：Edward 開綠界商店帳號 + 拿金鑰**）；沙利曼驗合規。
4. **Team seat**：主帳號 + seat 管理 + 權限 + 合併計費。
