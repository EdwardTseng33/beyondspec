## 三、🔥 最關鍵的 UX 風險（必須最優先）

### 🚨 Math.random() 造假風險（市場探測競爭矩陣）

**現況**：`renderFullReport` 內競爭矩陣仍用 `Math.random()` 生成 4 家競品的某些分數（v1.3.16 Hero + sparkline 重修時未觸及）。

**情境模擬**（P3 坤書）：
1. 坤書打開報告，看到競爭矩陣，打開 DevTools 看 data
2. 重新整理，發現同一家公司分數在變
3. 在 Twitter / Threads 發：「{@BeyondPath} 的競爭分析是亂數造的，居然敢收 Starter NT$299」
4. 其他 AI/技術型創辦人 retweet
5. **BeyondPath 信任破產**

**這不是假設**，這是 P3 持久行為模式的必然結果。

**立即修復方案**：
- **今晚必改**：移除 `Math.random()`，所有競爭矩陣分數改為「AI 推估 · 可變動」標籤，或直接拿掉未有實測支撐的欄位
- **中期方案**：接 DataForSEO 競品 serp 資料或 Ahrefs backlink 資料，真實計算

**工時**：移除 1 小時 + 標籤化 2 小時 = 3 小時 **（P0-最高優先）**

## 四、P0 必做（3 件）

### P0-1 · 移除 Math.random 假數據 ← 最高優先（上節）

### P0-2 · 資料源標記（AI 生成 vs 模板 vs 用戶輸入）

**現況**：用戶分不清報告裡哪個是「AI 講的」、哪個是「我填的」、哪個是「系統算的」。

**三色標記系統**（配合女巫 P1-2）：
- 🟢 **你的輸入** = 直接用你問卷或 PATH 答案
- 🟡 **系統推估** = 演算法或 AI 推論
- 🔴 **AI 生成** = 純 LLM 創作（如合成 persona、範例競品）

**效果**：P2 Tina 知道哪些可引用、P3 坤書滿意方法論、P4 美琪不被術語唬住。

**工時**：8 小時

---

### P0-3 · Lab Step A 引導 · 3 問題起手

**現況**：Lab 打開就是 4 個模組卡，P1 阿敏這類第一次用的人「不知從哪下手」。

**提案**：首次進 Lab 跳 3 問題小卡：
1. 「你想驗證什麼？」（產品方向 / 定價 / 行銷文案）
2. 「你有現成素材嗎？」（landing URL / PDF / 純想法）
3. 「你想花多久？」（10 分鐘速試 / 30 分鐘深入）

依答案 pre-select 模組 + pre-fill 範例輸入。

**工時**：6 小時

## 六、情緒溫度升級建議

**三層情緒目標**（Edward 要的 emotional design）：
1. **信任層**：P3 坤書打開報告不會想罵——靠「方法論透明 + 資料源標記 + Math.random 清零」
2. **包覆層**：P1/P4 打開不會茫然——靠「Executive One-Liner + Next Action + 術語翻譯」
3. **價值感層**：P5 打開覺得這份報告值錢——靠「視覺 hero + display 字階 + 儀式感節奏」

三層達成順序：**信任 → 包覆 → 價值感**（不能跳過信任直接做價值感）。

---

## 七、結論

**給 Edward 的 3 個優先判斷**：
1. **Math.random() 今晚必改**（不然 P3 風險隨時引爆）
2. **資料源三色標記系統**是整個報告升級的底層（所有 persona 都需要，投報比最高）
3. **Lab Step A 引導**比「問卷結果重修」更急（因 Lab 是新模組，onboarding 差會流失用戶）
